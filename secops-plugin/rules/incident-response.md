# Incident Response & Communication Rules

## MITRE ATT&CK

- ALWAYS map threats and incidents to MITRE ATT&CK framework when applicable.

## Vietnamese Regulatory Compliance

- ALWAYS check Vietnamese regulatory implications when dealing with:
  - Personal data (Nghị định 13/2023/NĐ-CP)
  - Information system security (Nghị định 85/2016/NĐ-CP)
  - Cybersecurity incidents (Luật An ninh mạng 2018)

## Vulnerability Analysis

- When analyzing vulnerabilities, consider exploitation status (CISA KEV), not just CVSS score.
- For risk assessments, always state both inherent risk AND residual risk (after controls).

## Communication

- Incident communications MUST include TLP marking: TLP:RED / TLP:AMBER / TLP:GREEN / TLP:WHITE.
- Never disclose vulnerability details publicly before remediation (responsible disclosure).
- Executive summaries should lead with business impact, not technical details.
- When in doubt about disclosure, escalate to legal/management — do not decide independently.
